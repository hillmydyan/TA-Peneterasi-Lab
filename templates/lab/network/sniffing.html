Network Sniffing Simulation

Apa itu Network Sniffing?

📝 Network Sniffing adalah proses memonitor dan menangkap semua lalu lintas data (packet traffic) yang lewat pada suatu jaringan komputer secara real-time. Teknik ini digunakan untuk menganalisis payload dan mendeteksi celah keamanan pada protokol yang tidak terenkripsi.

Aturan & Peringatan Etik

Segala bentuk simulasi dan perekaman traffic HANYA diizinkan di dalam ruang lingkup lab/web tertutup ini. ❌ Dilarang keras melakukan penyadapan jaringan pada public Wi-Fi atau jaringan yang bukan milik Anda!

Guide Instructions

Buka Practice Module

Informasi Login VM

Jika VM logout atau terkunci, gunakan kredensial berikut: Username: attacker | Password: admin
1

Start Virtual Machines

Nyalakan VM Attacker dan Victim. Pastikan keduanya berjalan lancar.

2

Config Network Mode

Pastikan kedua VM terhubung dalam satu jaringan internal (Host-Only / Internal Network) agar dapat saling berkomunikasi.

3

Buka Terminal

Silahkan buka terminal di VM Kali Linux Anda.

4

Identify Interface

Cek interface jaringan yang aktif pada Attacker VM:

ip a
5

Start Sniffing (Ettercap GUI)

Buka Ettercap dalam mode antarmuka grafis (GUI) dengan menjalankan perintah berikut:

sudo ettercap -G

Konfigurasi MITM / ARP Poisoning di GUI Ettercap:

  1. Pada jendela Setup, pilih Primary Interface (misal: eth0), lalu klik Centang (✓ Accept).
  2. Klik ikon Search (🔍) atau buka menu Hosts > Scan for hosts.
  3. Buka Hosts list (ikon bergambar daftar/list), lalu:
    • Pilih IP Gateway 192.168.56.111, lalu klik tombol Add to Target 1.
    • Pilih IP Target/Victim 192.168.56.104, lalu klik tombol Add to Target 2.
  4. Buka menu MITM (ikon bola dunia) > pilih ARP poisoning...
  5. Centang kotak "Sniff remote connections", lalu klik OK. Kini Ettercap sudah menyadap komunikasi IP target secara pasif.
6

Generate Traffic (Simulasi Login)

Di VM Target (IP 192.168.56.104), buka web browser dan kunjungi halaman login berikut:

http://192.168.56.111/login.php

Lakukan simulasi pengisian form login menggunakan kredensial berikut, lalu tekan tombol Login/Submit:

  • Username: admin
  • Password: 1234
7

Analyze Captured Credentials

Kembali ke layar VM Attacker dan perhatikan panel event log (pesan) di bagian bawah jendela Ettercap.

Karena website tersebut disajikan menggunakan protokol HTTP yang datanya tidak terkirim secara acak (tanpa enkripsi), Ettercap akan sukses menyadap post data sesi tersebut.

Anda akan mendeteksi teks USER: admin dan PASS: 1234 ditampilkan secara jelas di layar pemantauan!

8

Conclusion

Simpulkan informasi apa saja yang bisa didapatkan dari sniffing traffic yang tidak terenkripsi.
