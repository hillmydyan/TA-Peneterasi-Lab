DNS Spoofing Simulation

Apa itu DNS Spoofing?

📝 DNS Spoofing (atau DNS Cache Poisoning) adalah manipulasi informasi Domain Name System untuk mengalihkan lalu lintas korban ke server palsu. Target yang berniat mengakses web asli justru diarahkan ke web jebakan penyerang secara diam-diam.

Aturan & Peringatan Etik

Segala bentuk simulasi manipulasi lalu lintas jaringan HANYA diizinkan di dalam ruang lingkup lab/web tertutup ini. ❌ Dilarang keras membelokkan trafik atau merusak DNS pada jaringan publik!

Guide Instructions

Buka Practice Module

Informasi Login VM

Jika VM logout atau terkunci, gunakan kredensial berikut: Username: attacker | Password: admin
1

Start Virtual Machines

Nyalakan VM Attacker dan Victim. Pastikan keduanya berada dalam satu jaringan internal yang sama.

2

Buka Terminal

Silahkan buka terminal di VM Kali Linux Anda.

3

Network Check

Cek uji konektivitas ke Target/Victim. IP Target yang akan disimulasikan adalah 192.168.56.104.

ping 192.168.56.104
4

Konfigurasi DNS Ettercap

Sebelum memulai serangan berbasis GUI, edit konfigurasi bawaan DNS Ettercap untuk menambahkan pemetaan domain tujuan palsu ke alamat server Attacker (192.168.56.102):

# Buka file konfigurasi via Nano Editor:
sudo nano /etc/ettercap/etter.dns
# Tambahkan A Record ini di bagian paling bawah:
belajarit.test A 192.168.56.102

Simpan perubahan dan tutup file (tekan Ctrl+X, Y, lalu Enter pada terminal).

5

ARP Spoofing (Ettercap GUI)

Jalankan Ettercap dalam mode grafis (GUI) menggunakan perintah: sudo ettercap -G

Persiapan posisi Man-in-the-Middle:

  1. Pilih Primary Interface (misal: eth0), lalu klik tombol Centang (✓ Accept).
  2. Klik ikon Search (🔍) atau buka menu Hosts > Scan for hosts.
  3. Buka Hosts list: pilih IP Gateway 192.168.56.111 lalu terapkan sebagai Target 1. Kemudian, pilih IP Victim 192.168.56.104 lalu terapkan sebagai Target 2.
  4. Pilih menu MITM > klik ARP poisoning..., centang parameter "Sniff remote connections", lalu tekan OK.
6

Aktivasi Plugin DNS Spoof

Kini perangkat Anda sudah merepresentasikan saluran penghubung antara Victim dan dunia luar. Lakukan intersep pada permintaan web (DNS):

  • Buka menu dropdown di atas aplikasi: Plugins > Manage plugins.
  • Cari plugin yang bernama dns_spoof dalam daftar.
  • Klik dua kali (double-click) pada tulisan dns_spoof tersebut hingga muncul simbol tanda bintang (*) di sebelah sisinya.
  • Kini setiap kali Target menavigasi ke domain belajarit.test, Ettercap akan membengkokkannya secara rahasia.
7

Verification

Buka web browser di VM Victim dan kunjungi http://belajarit.test. Meskipun Anda mengetikkan domain asli, halaman yang terbuka adalah halaman server lokal milik Attacker (192.168.56.102) akibat manipulasi DNS.
